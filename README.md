# AeroTunnel

**Διαδραστικός Προσομοιωτής Υπολογιστικής Ρευστοδυναμικής (CFD) στον Browser**

Ένα πλήρως client-side εργαλείο αεροδυναμικής ανάλυσης που τρέχει σε πραγματικό χρόνο μέσα στον browser, χωρίς backend ή εξωτερικούς υπολογισμούς. Βασίζεται στη μέθοδο Lattice Boltzmann (LBM) με πλέγμα D2Q9 και σύγκρουση BGK.

**Live Demo:** [aerotunnel](https://georgiosbalatzis.github.io/aerotunnel/)

## Επισκόπηση

Το AeroTunnel είναι ένας ολοκληρωμένος προσομοιωτής αεροδυναμικής σήραγγας που εκτελείται εξ ολοκλήρου στον browser. Απευθύνεται σε μηχανικούς, φοιτητές και λάτρεις της αεροδυναμικής — ιδιαίτερα της Formula 1 — που θέλουν να πειραματιστούν με ροές γύρω από διάφορα αεροδυναμικά προφίλ σε πραγματικό χρόνο.

Η εφαρμογή λύνει τις εξισώσεις ροής χρησιμοποιώντας τη μέθοδο **Lattice Boltzmann** αντί των παραδοσιακών εξισώσεων Navier-Stokes. Αυτή η προσέγγιση είναι ιδανική για παράλληλους υπολογισμούς και παρέχει εξαιρετική απόδοση σε JavaScript, επιτρέποντας προσομοίωση σε πραγματικό χρόνο χωρίς GPU computing ή server-side επεξεργασία.

## Θεωρητικό Υπόβαθρο

### Lattice Boltzmann Method (LBM)

Η μέθοδος Lattice Boltzmann (LBM) αποτελεί μια εναλλακτική προσέγγιση στην υπολογιστική ρευστοδυναμική. Αντί να επιλύει απευθείας τις εξισώσεις Navier-Stokes (συνέχεια, ορμή, ενέργεια), η LBM μοντελοποιεί τη ροή σε μεσοσκοπικό επίπεδο μέσω **συναρτήσεων κατανομής** (distribution functions).

Η βασική ιδέα: αντί να παρακολουθούμε κάθε μόριο ρευστού ξεχωριστά ή να λύνουμε μακροσκοπικές εξισώσεις, χρησιμοποιούμε στατιστικές κατανομές σωματιδίων πάνω σε ένα κανονικό πλέγμα. Σε κάθε χρονικό βήμα εκτελούνται δύο λειτουργίες:

1. **Streaming (Ροή):** Οι κατανομές μεταφέρονται στους γειτονικούς κόμβους κατά μήκος των διακριτών ταχυτήτων του πλέγματος.
2. **Collision (Σύγκρουση):** Οι κατανομές χαλαρώνουν προς την τοπική κατανομή ισορροπίας, προσομοιώνοντας τις μοριακές συγκρούσεις.

Μαθηματικά, η εξίσωση Lattice Boltzmann εκφράζεται ως:

```
f_i(x + c_i·Δt, t + Δt) = f_i(x, t) + Ω_i(x, t)
```

όπου `f_i` είναι η συνάρτηση κατανομής στην κατεύθυνση `i`, `c_i` ο διακριτός διανυσματικός ρυθμός, και `Ω_i` ο τελεστής σύγκρουσης.

### Πλέγμα D2Q9

Το AeroTunnel χρησιμοποιεί πλέγμα **D2Q9** — δηλαδή 2 Διαστάσεις με 9 Ταχύτητες ανά κόμβο. Κάθε κόμβος του πλέγματος έχει 9 κατευθύνσεις κίνησης:

```
  6   2   5
    \ | /
  3 - 0 - 1
    / | \
  7   4   8
```

- **Κατεύθυνση 0:** Ακίνητη (c = [0, 0])
- **Κατευθύνσεις 1-4:** Κύριοι άξονες (οριζόντια/κατακόρυφα)
- **Κατευθύνσεις 5-8:** Διαγώνιες

Οι αντίστοιχες σταθερές στον κώδικα:

| Σταθερά | Τιμές | Περιγραφή |
|---------|-------|-----------|
| `CX` | `[0, 1, 0, -1, 0, 1, -1, -1, 1]` | Συνιστώσα x της ταχύτητας πλέγματος |
| `CY` | `[0, 0, 1, 0, -1, 1, 1, -1, -1]` | Συνιστώσα y της ταχύτητας πλέγματος |
| `WT` | `[4/9, 1/9, ..., 1/36, ...]` | Βάρη ισορροπίας (weights) |
| `OPP` | `[0, 3, 4, 1, 2, 7, 8, 5, 6]` | Αντίθετη κατεύθυνση (για bounce-back) |

Τα βάρη `WT` προκύπτουν από τη θεωρία κινητικής: 4/9 για την ακίνητη κατεύθυνση, 1/9 για τους κύριους άξονες, και 1/36 για τις διαγώνιες.

### Σύγκρουση BGK

Ο τελεστής σύγκρουσης **BGK** (Bhatnagar-Gross-Krook) είναι η πιο διαδεδομένη απλοποίηση του τελεστή σύγκρουσης:

```
Ω_i = ω · (f_i^eq - f_i)
```

όπου:
- `ω` (omega) είναι η **παράμετρος χαλάρωσης** (relaxation parameter), που συνδέεται με το ιξώδες
- `f_i^eq` είναι η κατανομή ισορροπίας Maxwell-Boltzmann

Η κατανομή ισορροπίας υπολογίζεται ως:

```
f_i^eq = w_i · ρ · (1 + 3·(c_i · u) + 4.5·(c_i · u)² - 1.5·|u|²)
```

Η σχέση μεταξύ ω και κινηματικού ιξώδους ν:

```
ω = 1 / (3·ν + 0.5)
```

Στον κώδικα, η τιμή ω περιορίζεται στο εύρος [0.5, 1.95] για αριθμητική ευστάθεια. Τιμές κοντά στο 2.0 αντιστοιχούν σε πολύ χαμηλό ιξώδες (υψηλός αριθμός Reynolds) αλλά μπορεί να προκαλέσουν αστάθεια.

### Συνοριακές Συνθήκες

Η προσομοίωση εφαρμόζει τέσσερα είδη συνοριακών συνθηκών:

#### 1. Είσοδος — Zou-He Boundary Condition
Στην αριστερή πλευρά του πεδίου εφαρμόζεται η συνοριακή συνθήκη **Zou-He**, η οποία επιβάλλει μια καθορισμένη ταχύτητα εισόδου (`inU`) ενώ υπολογίζει τις κατανομές με τρόπο που εξασφαλίζει συνέπεια μεταξύ μακροσκοπικών και μικροσκοπικών μεγεθών. Προαιρετικά προστίθεται μικρή τυχαία διαταραχή (`turb`) για την πυροδότηση τυρβώδους συμπεριφοράς.

#### 2. Έξοδος — Zero-gradient (Copy)
Στη δεξιά πλευρά, οι κατανομές αντιγράφονται από τον προηγούμενο κόμβο (extrapolation), επιτρέποντας στη ροή να εξέρχεται ελεύθερα χωρίς ανακλάσεις.

#### 3. Τοίχοι — No-slip (Bounce-back)
Στο πάνω και κάτω όριο εφαρμόζεται η συνθήκη **bounce-back**: οι κατανομές που φτάνουν στον τοίχο αντιστρέφονται στην αντίθετη κατεύθυνση, δημιουργώντας μηδενική ταχύτητα στην επιφάνεια (no-slip condition).

#### 4. Στερεά Σώματα — Full-way Bounce-back
Τα αεροδυναμικά σώματα υλοποιούνται ως solid nodes μέσω ενός Uint8Array. Σε κάθε solid κόμβο, οι κατανομές αναστρέφονται πλήρως: `f_i ↔ f_opp(i)`. Αυτό δημιουργεί αυτόματα μηδενική ταχύτητα στην επιφάνεια και επιτρέπει τη μοντελοποίηση σωμάτων αυθαίρετου σχήματος μέσω αλγορίθμου **point-in-polygon** (ray casting).

### Μακροσκοπικά Μεγέθη

Από τις συναρτήσεις κατανομής υπολογίζονται τα φυσικά μεγέθη:

| Μέγεθος | Τύπος | Περιγραφή |
|---------|-------|-----------|
| **Πυκνότητα (ρ)** | `ρ = Σ f_i` | Άθροισμα όλων των κατανομών στον κόμβο |
| **Ταχύτητα (u)** | `u = (Σ c_i·f_i) / ρ` | Μέση ταχύτητα ρευστού |
| **Μέτρο ταχύτητας** | `\|u\| = √(ux² + uy²)` | Για χρωματική απεικόνιση |
| **Στροβιλότητα (curl)** | `ω_z = ∂uy/∂x - ∂ux/∂y` | Υπολογίζεται με κεντρικές διαφορές |

Η **πίεση** υπολογίζεται από την καταστατική εξίσωση του πλέγματος: `p = ρ·c_s²` όπου `c_s = 1/√3` η ταχύτητα ήχου στο πλέγμα.

## Αρχιτεκτονική

### Μηχανή Προσομοίωσης (Engine)

Η μηχανή βρίσκεται στον φάκελο `src/engine/` και αποτελείται από:

**`solver.js` — Κλάση LBM**
Ο πυρήνας της προσομοίωσης. Η κλάση `LBM` διαχειρίζεται:
- Δύο buffers κατανομών (`f0`, `f1`) τύπου `Float32Array` μεγέθους 9×N, όπου N = COLS × ROWS
- Πεδία `rho`, `ux`, `uy`, `spd`, `curl` για τα μακροσκοπικά μεγέθη
- Πεδίο `solid` (Uint8Array) για τον ορισμό στερεών σωμάτων
- Παράμετρο χαλάρωσης `omega`

Η μέθοδος `step()` εκτελεί ένα πλήρες χρονικό βήμα:
1. Streaming (μεταφορά κατανομών)
2. Bounce-back σε στερεά
3. Υπολογισμός μακροσκοπικών μεγεθών
4. Εφαρμογή Zou-He στην είσοδο
5. Zero-gradient στην έξοδο
6. No-slip σε πάνω/κάτω τοίχους
7. Σύγκρουση BGK
8. Υπολογισμός στροβιλότητας

**`constants.js` — Σταθερές Πλέγματος**
Ορίζει τις διαστάσεις της προσομοίωσης. Σε desktop: 300×135 κόμβοι (40.500 κελιά). Σε mobile: 208×104 (21.632 κελιά) για καλύτερη απόδοση.

**`particles.js` — Σύστημα Σωματιδίων**
Βλέπε παρακάτω.

**`geometry.js` — Γεωμετρικοί Μετασχηματισμοί**
Βλέπε παρακάτω.

**`parsers.js` — Αναγνώστες Αρχείων**
Βλέπε παρακάτω.

**`lut.js` — Πίνακες Χρωμάτων**
Δύο colormaps: **Turbo** (πολύχρωμος, για ταχύτητα) και **CoolWarm** (μπλε-κόκκινο, για πίεση/στροβιλότητα).

### Σύστημα Σωματιδίων

Τα σωματίδια χρησιμοποιούνται για την οπτικοποίηση ροϊκών γραμμών (streamlines). Κάθε σωματίδιο:

- Εκπέμπεται από την αριστερή πλευρά του πεδίου
- Μετακινείται ακολουθώντας το πεδίο ταχυτήτων μέσω ολοκλήρωσης **Runge-Kutta 2ης τάξης** (μέθοδος μέσου σημείου)
- Αποθηκεύει ιστορικό θέσεων (trail) σε κυκλικό buffer μήκους 80 (ή 56 στο mobile)
- Επαναφέρεται (reset) όταν βγει εκτός ορίων ή χτυπήσει στερεό

Η ταχύτητα στη θέση κάθε σωματιδίου υπολογίζεται με **διγραμμική παρεμβολή** (bilinear interpolation) μεταξύ των τεσσάρων γειτονικών κόμβων, εξασφαλίζοντας ομαλές τροχιές.

Το pool σωματιδίων προκαθορίζεται (max 2000 στο desktop, 900 στο mobile) και ο χρήστης μπορεί να ρυθμίσει πόσα είναι ενεργά.

### Γεωμετρία και Parsers

**Γεωμετρικοί Μετασχηματισμοί:**
- `normPoly`: Κανονικοποιεί ένα πολύγωνο στο εύρος [0,1]×[0,1]
- `xformPoly`: Μετατόπιση, κλιμάκωση και περιστροφή στο πεδίο LBM
- `simplPoly`: Απλοποίηση πολυγώνου (Douglas-Peucker style) με ανοχή

**Προκαθορισμένα Προφίλ:**
Η εφαρμογή περιλαμβάνει 7 έτοιμα αεροδυναμικά σχήματα, χωρισμένα σε δύο κατηγορίες:

| Κατηγορία | Προφίλ | Περιγραφή |
|-----------|--------|-----------|
| **Benchmark** | Airfoil | Κλασικό αεροτομικό προφίλ με κυρτότητα (camber) |
| | Cylinder | Κύλινδρος — κλασικό benchmark για ανίχνευση ομόρροιας Von Kármán |
| | Wedge | Σφήνα — μελέτη αιχμηρής ακμής πρόσπτωσης |
| | Bluff | Ορθογωνικό σώμα — μελέτη αποκόλλησης ροής |
| **F1 Profiles** | F1 Car | Πλήρης σιλουέτα μονοθεσίου Formula 1 |
| | Front Wing | Εμπρόσθια αεροτομή — downforce στον πρόσθιο άξονα |
| | Rear Wing | Οπίσθια πτέρυγα — υψηλό downforce |

**Εισαγωγή Αρχείων:**
Ο χρήστης μπορεί να εισάγει δικά του σχήματα μέσω:
- **SVG:** Υποστηρίζει `<path>`, `<polygon>`, `<polyline>`, `<rect>`, `<circle>`, `<ellipse>`. Τα paths δειγματοληπτούνται μέσω `getPointAtLength()`.
- **DXF:** Εξάγει σημεία (group codes 10/20) από αρχεία AutoCAD.
- **STL:** Υποστηρίζει ASCII και binary STL — εξάγει τομή στο μέσο Z.
- **Image (PNG/JPG):** Ανίχνευση ακμών με threshold φωτεινότητας και ανίχνευση γειτόνων.
- **Sketch:** Ελεύθερο σχέδιο στο canvas με ποντίκι/αφή.

### Οπτικοποίηση

Η εφαρμογή προσφέρει **5 τρόπους απεικόνισης**:

| Λειτουργία | Περιγραφή |
|------------|-----------|
| **Velocity** | Χρωματική απεικόνιση μέτρου ταχύτητας (colormap Turbo) |
| **Pressure** | Χρωματική απεικόνιση πίεσης (colormap CoolWarm) |
| **Streamlines** | Τροχιές σωματιδίων — ημιδιαφανή trails με fade-out |
| **Vorticity** | Στροβιλότητα (curl) — αναδεικνύει δίνες και αποκόλληση |
| **3D View** | Τρισδιάστατη εξώθηση (extrusion) του προφίλ με χρωματισμό επιφάνειας μέσω Three.js |

Η 2D απεικόνιση χρησιμοποιεί **Canvas 2D API** με `ImageData` και `Uint32Array` buffer για άμεσο pixel manipulation. Κάθε κόμβος αντιστοιχίζεται σε pixel, η τιμή κανονικοποιείται, και αναζητείται στον πίνακα χρωμάτων (LUT) 256 εισόδων.

Η 3D απεικόνιση φορτώνει δυναμικά το **Three.js r128** από CDN, δημιουργεί εξωθημένη γεωμετρία από το 2D πολύγωνο, και εφαρμόζει vertex colors από τα δεδομένα του solver. Περιλαμβάνει orbit controls, fog, πολλαπλά φώτα, και grid helper.

## Δυνατότητες

### Μηχανή Ρευστοδυναμικής
- Lattice Boltzmann D2Q9 σε πραγματικό χρόνο
- Ρυθμιζόμενη ταχύτητα εισόδου (0.02 - 0.18 lattice units)
- Ρυθμιζόμενη ένταση τύρβης (0 - 3)
- Ρυθμιζόμενο κινηματικό ιξώδες (0.005 - 0.1)
- Ταχύτητα προσομοίωσης 1x - 8x

### Αεροδυναμικά Εργαλεία
- 7 προκαθορισμένα προφίλ (benchmark + F1)
- Εισαγωγή SVG, STL, DXF, εικόνας
- Ελεύθερο σχέδιο (sketch)
- Μετατόπιση, κλιμάκωση, περιστροφή (γωνία πρόσπτωσης -25° έως +35°)
- Απλοποίηση πολυγώνου (polygon simplification)

### Ζωντανές Μετρήσεις
- **CL** (Lift Coefficient): Συντελεστής άνωσης — υπολογίζεται από τη διαφορά πίεσης πάνω/κάτω
- **CD** (Drag Coefficient): Συντελεστής οπισθέλκουσας — αντίσταση στη ροή
- **Re** (Reynolds Number): Αριθμός Reynolds — λόγος αδρανειακών/ιξωδών δυνάμεων
- **L/D** (Lift-to-Drag Ratio): Αεροδυναμική αποδοτικότητα
- Sparklines ιστορικού μετρήσεων
- Ταξινόμηση καθεστώτος ροής (Laminar / Transitional / Turbulent)

### Αναφορές Σύγκρισης
Ενσωματωμένα δεδομένα αναφοράς για σύγκριση:
- NACA 0012: CL 0.00-1.20, CD 0.006-0.020
- NACA 2412: CL 0.30-1.60, CD 0.006-0.020
- F1 rear wing: CL 2.50+, CD 0.80-1.20
- Flat plate: CL 0.00-1.00, CD 0.02-1.28

### Περιβάλλον Χρήστη
- Σκούρο/φωτεινό θέμα
- Responsive design (desktop + mobile)
- Command bar (/) για γρήγορη πλοήγηση
- Πληκτρολογικές συντομεύσεις (Space, R, 1-5, F, S, Z, /)
- Icon rail πλευρικής πλοήγησης
- Fullscreen mode
- Snapshot (λήψη στιγμιοτύπου)
- Undo shape history
- Panel ανάλυσης με γραφήματα sparkline
- Panel πληροφοριών (About)

## Εγκατάσταση και Εκκίνηση

```bash
# Κλωνοποίηση
git clone https://github.com/georgiosbalatzis/aerotunnel.git
cd aerotunnel

# Εγκατάσταση εξαρτήσεων
npm install

# Εκκίνηση development server
npm run dev
```

## Χρήση

### Βασική Ροή Εργασίας

1. **Επιλέξτε σχήμα** — Από τα presets (Airfoil, Cylinder, F1 Car κ.λπ.) ή εισάγετε δικό σας αρχείο.
2. **Ρυθμίστε τη θέση** — Μετακινήστε, κλιμακώστε και περιστρέψτε το σχήμα στο πεδίο ροής.
3. **Ρυθμίστε τη ροή** — Αλλάξτε ταχύτητα εισόδου, ιξώδες και τύρβη.
4. **Εκκινήστε** — Πατήστε Space ή ενεργοποιήστε το Auto Green-Flag.
5. **Παρατηρήστε** — Εναλλάξτε μεταξύ Velocity, Pressure, Streamlines, Vorticity, 3D.
6. **Αναλύστε** — Ελέγξτε CL, CD, Re στο panel μετρήσεων.

### Πληκτρολογικές Συντομεύσεις

| Πλήκτρο | Ενέργεια |
|---------|----------|
| `Space` | Εκκίνηση / Παύση |
| `R` | Επαναφορά solver |
| `1` - `5` | Εναλλαγή τρόπου απεικόνισης |
| `F` | Fullscreen |
| `S` | Λήψη στιγμιοτύπου (snapshot) |
| `Z` | Αναίρεση σχήματος |
| `/` | Εμφάνιση πληκτρολογικών συντομεύσεων |

## Αριθμητικές Λεπτομέρειες

### Μέγεθος Πλέγματος και Απόδοση

Σε κάθε χρονικό βήμα, ο solver εκτελεί τους εξής υπολογισμούς:

- **Streaming:** 9 αναγνώσεις + 9 εγγραφές ανά κόμβο = ~729.000 λειτουργίες (desktop)
- **Collision:** 9 εκτιμήσεις ισορροπίας + 9 ενημερώσεις ανά κόμβο
- **Μακροσκοπικά:** 1 πέρασμα για ρ, ux, uy, speed
- **Στροβιλότητα:** 1 πέρασμα με κεντρικές διαφορές

Σύνολο: ~6 περάσματα πάνω από το πλέγμα ανά βήμα. Σε ταχύτητα 8x, εκτελούνται 8 βήματα ανά frame (≈60fps), δηλαδή ~48 πλήρη περάσματα/frame.

### Αριθμητική Ευστάθεια

Ο solver χρησιμοποιεί μερικές τεχνικές ασφαλείας:
- **Clamping πυκνότητας:** Αν `ρ < 0.01`, επαναφέρεται σε 1 (αποφυγή διαίρεσης με μηδέν)
- **Clamping omega:** Περιορίζεται στο [0.5, 1.95] (αποφυγή αρνητικού ιξώδους)
- **Turbulence perturbation:** Μικρές τυχαίες διαταραχές (`±0.004×turb`) στην είσοδο

### Μονάδες Πλέγματος (Lattice Units)

Όλα τα μεγέθη εκφράζονται σε **lattice units**:
- Μονάδα μήκους: 1 κελί πλέγματος (Δx = 1)
- Μονάδα χρόνου: 1 βήμα (Δt = 1)
- Ταχύτητα ήχου: c_s = 1/√3 ≈ 0.577
- Ταχύτητα ροής: τυπικά 0.02 - 0.18 (< c_s για συμπιεστότητα < 10%)

Ο αριθμός Mach (Ma = u/c_s) πρέπει να παραμένει χαμηλός (< 0.3) για να ισχύει η ασυμπίεστη προσέγγιση της LBM.
